Page Turners

An application for readers to track their current book collection and unreads. A group programming React application that uses the Google Books restful API to allow users to search by title or author, using separate api calls for either

Users can add their finds to a reading list that is stored in a Firebase Realtime Database and that information is stored on the "My Reading List" that changes live as the reader removes, adds, and sets books as read. User also has a progress bar that shows them their reading progress through their collection.

TechOccupy

An application for job hunters in the tech industry, users can search by job field, experience level and choose 1 of 4 popular tech cities.

Users make their selection and submit their form which creates a promise and filters out results that fit their search parameters. Results from API display the job description from the API, locations available to work include remote availability, and an active link that takes users to the job board posting for users to apply to the position.

Where's That Pokémon?

Where's That Pokémon?

jQuery project that allows the user to search for any Pokémon from the existing database.

Results of the search list the statistics of the Pokémon, a sprite, weaknesses and an entire list of locations found for that Pokémon across the entirety of the video game series.

This search engine required two different ajax calls as the API had information separated at alternate endpoints. There are also ID endpoints that required error handling because they have no locations available.

NutriCo.

React project that incorporates a nutrition API that allows the user to search for grocery products and simple foods. Users can select the items to view their nutritional info through a second API call, and add the items to their Journal

The journal is connected to Firebase through an OnValue realtime database. The database is split into two categories as the API objects are shaped differently depending on the product.

React Application

The very first React project I ever completed, using a nutritional API to track and log the user's daily diet and compile macronutrients. Firebase was an optional task but I believed it was essential for the functionality. Users can save every item in their daily diet and view their complete nutritional log for the day. Items are split into simple foods and grocery products as these were the most detailed components of the API used.
